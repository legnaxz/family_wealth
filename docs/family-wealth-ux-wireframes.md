# Family Wealth MVP — UX/UI 확장 문서 + 와이어프레임

## 목적

현재 `family-wealth-mvp`를 다음 구조로 확장한다.

- **전체 페이지**: 부부 전체 데이터 통합 뷰
- **본인 페이지**: 개인 데이터 전용 뷰
- **배우자 페이지**: 배우자 데이터 전용 뷰

핵심 원칙은 다음과 같다.

1. **웹은 전체 기록과 분석을 보여준다**
2. **모바일 웹은 지금 꼭 필요한 정보만 보여준다**
3. **개인/배우자/전체 컨텍스트가 절대 헷갈리면 안 된다**
4. **향후 배우자 데이터 import를 고려해 owner/person 축을 먼저 설계한다**

---

## 제품 구조 제안

### 정보 구조

#### 1) 전체
- 부부 합산 월 수입/지출/순현금흐름
- 자산/부채 통합 현황
- 사람별 지출 비교
- 카테고리별 합산
- 최근 거래 전체 로그

#### 2) 본인
- 개인 수입/지출/순현금흐름
- 개인 자산/부채
- 개인 카테고리별 소비
- 개인 거래 목록

#### 3) 배우자
- 배우자 수입/지출/순현금흐름
- 배우자 자산/부채
- 배우자 카테고리별 소비
- 배우자 거래 목록

### 데이터 ownership 원칙

모든 거래/자산/부채는 아래 owner 중 하나를 가진다.

- `self`
- `spouse`
- `shared`

페이지별 계산 방식:

- **전체** = `self + spouse + shared`
- **본인** = `self`
- **배우자** = `spouse`

공동 생활비나 공동 카드가 생기면 `shared`가 중요해진다.

## 웹 vs 모바일 역할 분리

### 웹
웹은 운영/분석 도구에 가깝다.

보여줘야 하는 것:
- 월/주/일 필터
- 거래 검색
- 카테고리 분석
- 사람별 비교
- 자산/부채 추이
- import 로그
- 거래 수정/정리

### 모바일 웹
모바일은 확인/반응 도구에 가깝다.

보여줘야 하는 것:
- 이번 달 얼마나 썼는지
- 이번 달 수입/지출/순현금흐름
- 최근 거래
- 오늘/이번 주 특이사항
- 누구 기준 화면인지(전체/본인/배우자)
- 빠른 정리 액션

모바일에서 불필요하게 복잡한 분석 차트는 뒤로 빼는 게 맞다.

## UX 원칙

### 1. 컨텍스트를 항상 상단에 고정
사용자는 지금 어떤 관점의 데이터를 보는지 항상 알아야 한다.

추천 UI:
- `전체 | 본인 | 배우자`

모바일/웹 모두 상단 세그먼트 또는 탭으로 제공한다.

### 2. 모바일은 요약 → 인사이트 → 최근 기록 순서
상단부터 아래 순서:
1. 이번 달 요약
2. 이상치/인사이트
3. 최근 거래
4. 달력/리스트 탐색

### 3. 달력은 탐색용, 상세는 리스트/바텀시트로
달력 칸에 너무 많은 숫자를 넣지 않는다.

달력 셀에는:
- 날짜
- 순유출/순유입 핵심 숫자 하나
- 수입/지출 존재 여부 정도만 표시

탭하면 아래 상세 패널에서 거래를 보여준다.

### 4. 거래 화면은 단순 조회가 아니라 정리 가능한 화면이어야 함
각 거래에는 최소한 아래 액션이 필요하다.

- 카테고리 수정
- 메모 추가
- 제외 처리
- 중복/오류 표시
- owner 변경 (`self / spouse / shared`)

## 배우자 데이터 유입 이후 동작 원칙

### import 규칙
- 본인 계정 import 시 `owner=self`
- 배우자 계정 import 시 `owner=spouse`
- 공동 계좌/공동 카드가 있다면 별도 규칙으로 `owner=shared`

### 집계 규칙
- 전체 페이지는 세 owner를 합산
- 개인 페이지는 자기 owner만 표시
- 향후 `shared`를 개인 페이지에 어떻게 배분해서 보여줄지는 정책 필요
  - 기본안: 개인 페이지에서는 `shared` 제외
  - 대안: shared 별도 섹션 표기

## UI 우선순위 정리

### 모바일에서 꼭 보여야 하는 것
1. 현재 보고 있는 대상(전체/본인/배우자)
2. 이번 달 지출/수입/순현금흐름
3. 최근 거래
4. 특이사항
5. 빠른 수정 액션

### 모바일에서 뒤로 빼도 되는 것
- 큰 멀티 시리즈 차트
- 복잡한 비교 테이블
- 너무 상세한 카테고리 breakdown
- 과도한 숫자 밀도의 달력 셀

### 웹에서 강화할 것
- 사람별 비교
- 자산/부채 추이
- 검색/필터/수정
- import 품질 관리

## 결론

이 제품은 단순 가계부보다 **가족 단위 재무 운영 도구**에 가깝게 가는 게 맞다.

가장 중요한 건 세 가지다.

1. **owner/person 축을 먼저 심는다**
2. **모바일은 요약 중심으로 단순화한다**
3. **전체 / 본인 / 배우자 컨텍스트를 강하게 드러낸다**

이 원칙만 지키면,
현재 구조에서 부부 확장과 모바일 최적화를 동시에 가져갈 수 있다.
